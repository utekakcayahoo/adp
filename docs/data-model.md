# Data Model — `main.adp`

Five tables, generated by `data/adp_data_generator.py` (deployed as the
`adp_data_generator` Databricks Job). **Emissions are never stored** — they're
computed downstream as `kWh × emission factor`.

## Tables

### `adp_facilities` — the buildings (1 row / facility)
| column | type | notes |
|---|---|---|
| facility_id | string | `FAC-001` … `FAC-006` |
| name | string | display name |
| type | string | `office` \| `warehouse` \| `datacenter` |
| city | string | Seattle / Chicago / Austin |
| grid_region | string | `US-NW` \| `US-MW` \| `US-TX` (joins to factors) |
| floor_area_sqm | int | size driver for base load |
| opened_on | date | |

### `adp_emission_factors` — kgCO₂e per kWh (1 row / grid_region+fuel)
| column | type | notes |
|---|---|---|
| grid_region | string | |
| fuel | string | `electricity` \| `gas` |
| kgco2e_per_kwh | double | electricity: NW 0.20 / MW 0.45 / TX 0.38; gas 0.18 |
| valid_from / valid_to | date | factor validity window |
| source | string | **`illustrative`** — plausible, not certified |

### `adp_targets` — reduction goals (org + per-facility)
| column | type | notes |
|---|---|---|
| scope | string | `org` \| `facility` |
| facility_id | string (nullable) | null for the org row |
| baseline_year | int | 2024 |
| baseline_tco2e | double | **computed from real 2024 generated data** |
| target_year | int | 2030 |
| target_reduction_pct | double | org = 50; facilities vary 40–55 |

### `adp_weather` — drives consumption (hourly / city)
| column | type | notes |
|---|---|---|
| weather_ts | timestamp | hourly |
| city | string | |
| temp_c | double | seasonal + daily cycle + noise |
| heating_degree_hours | double | `max(0, 18 − temp)` |
| cooling_degree_hours | double | `max(0, temp − 22)` |

### `adp_energy_readings` — the fact table (hourly / facility)
| column | type | notes |
|---|---|---|
| reading_ts | timestamp | hourly |
| facility_id | string | |
| electricity_kwh | double | base × shape × eff + cooling + anomalies + noise |
| gas_kwh | double | base × shape × eff + heating + noise |
| source | string | `meter` |

History starts **2024-01-01**; the scheduled `append` run keeps it current.

## Knowledge corpus — `adp_standards` (Phase 5, RAG)
Not produced by the data job — seeded by `data/seed_standards.py`. Holds the written
policies the co-pilot cites, so recommendations and policy answers are grounded in text
rather than invented.

| column | type | notes |
|---|---|---|
| id | string | PK, e.g. `STD-HVAC-SETPOINT` |
| title | string | short policy title |
| category | string | hvac / operations / accounting / targets / efficiency / datacenter / reporting / warehouse / procurement |
| body | string | the policy text (this is what gets embedded) |
| source | string | illustrative source label |

Change Data Feed is enabled (required for the Delta Sync index). A Databricks **Vector
Search** index `main.adp.adp_standards_index` (endpoint `adp_vs`, Databricks-managed
embeddings via `databricks-gte-large-en`) indexes `body` and is exposed to the agent as
the `search_standards` tool. **Billable:** the `adp_vs` endpoint accrues cost while it
exists — delete it (`databricks vector-search-endpoints delete-endpoint adp_vs`) when the
phase is done.

## Answer key — planted anomalies (for testing the Diagnose capability)
> The agent should *discover* these from the data; this is our ground truth to
> check it against. Do not feed this to the agent.

1. **FAC-004 (Central Warehouse) — HVAC fault.** Electricity ×1.8 from
   **2025-03-05 to 2025-04-20**. A sharp, bounded spike.
2. **FAC-006 (Edge Data Center) — load creep.** Electricity grows ~6%/yr,
   outrunning the ~3%/yr efficiency trend — a slow, persistent upward drift.

Everything else trends mildly **down** (~3%/yr efficiency gain), so the org is
*partly* on track toward its 2030 target — making "are we on track?" a real question.
